1. Executive Summary

This report presents a locally hosted multi-agent system (MAS) that automates bug analysis, root cause detection, patch generation, validation, and reporting for software projects.

2. Problem Definition and Domain

Domain: Software Engineering / Internal Developer Tooling

Objective: Automatically transform an issue signal into a validated patch and explainable report.

3. System Architecture

Core Components

  • Frontend Console (input + results)
  • FastAPI backend orchestration
  • LangGraph stateful workflow
  • Ollama-hosted local LLM
  • Tooling layer (search, patch, test, report)

Execution Properties

  • Local-only model execution
  • Structured state handoff across agents
  • Artifact generation (patch + markdown report)
  • Rollback support on failed validation

4. Agent Design and Responsibilities

Agent Input Output Primary Tools
Classifier Bug report Bug type, severity, summary Bug report parser
Root Cause Classification + project path Suspected file/function + evidence Code search, risk scanner
Fix Generator Root cause + source file Patch + diff + backup Patch writer, backup tool
Tester Patched workspace Validation result + report Test runner, report writer

5. Tools and Environment Interaction

  • Reads and scans source files by language.
  • Applies patches and writes backup artifacts.
  • Runs language-specific tests or fallback validation checks.
  • Emits trace logs and downloadable artifacts.

6. Workflow and Shared State

received -> classified -> analyzed -> patched -> tested/rolled_back

Key state fields: run_id, bug_report, project_path, language, classification, root_cause, patch, test_results, final_summary.

7. Implementation Details

Local Deployment

pip install -e .
python -m uvicorn bug_fixing_mas.api:app --reload

Autonomy Improvements

  • Low-signal prompts are augmented with static bug hints from code.
  • Fallback validation executes compile/syntax checks when tests are missing.
